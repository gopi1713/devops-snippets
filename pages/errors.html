Common Errors & Solutions

Find quick solutions to frequently encountered issues in development environments

Magento Common Errors

PHP Extension intl Missing

When installing Magento, you receive an error about missing PHP extension

PHP Warning: Magento supports PHP 7.0.2 or newer | PHP Extension "intl" must be installed
Solution

Install the missing PHP extension with the package manager:

For Ubuntu/Debian: sudo apt-get install php-intl

For CentOS/RHEL: sudo yum install php-intl

Missing BCMath Extension

During Magento 2 installation, PHP reports missing bcmath extension

Check for required PHP extensions (php -m): 'bcmath' must be installed
Solution

Install the BCMath PHP extension:

For Ubuntu/Debian: sudo apt-get install php-bcmath

For CentOS/RHEL: sudo yum install php-bcmath

Slow Admin Panel

The Magento Admin panel is extremely slow

The admin panel takes 30+ seconds to load simple pages
Solution

1. Enable caching for admin: bin/magento cache:enable

2. Move to production mode: bin/magento deploy:mode:set production

3. Check for poorly performing third-party modules

Static Content Deploy Timeout

Static content deployment takes too long or times out

ERROR: Command execution timeout after 900 seconds
Solution

Use these flags for faster deployment:

bin/magento setup:static-content:deploy -f --no-interaction --jobs=4

Deploy only necessary languages:

bin/magento setup:static-content:deploy en_US -f

Permission Issues

Permission denied errors during deployment

Warning: file_put_contents(): Failed to open stream: Permission denied
Solution

Set proper permissions:

find var generated vendor pub/static pub/media app/etc -type f -exec chmod g+w {} +

find var generated vendor pub/static pub/media app/etc -type d -exec chmod g+ws {} +

chmod u+x bin/magento

Cache Clean Fails

Unable to clean cache

Cache types config flushing failed. Please run "setup:upgrade"
Solution

Try manual cache clearing:

rm -rf var/cache/* var/page_cache/* generated/code/*

bin/magento setup:upgrade

bin/magento cache:flush

MySQL Common Errors

Access Denied for User

MySQL user authentication error, preventing access to the database

ERROR 1045 (28000): Access denied for user 'username'@'localhost' (using password: YES)
Solution

1. Check if username and password are correct

2. Reset the password if necessary:

ALTER USER 'username'@'localhost' IDENTIFIED BY 'new_password';

3. Verify the user has proper permissions:

SHOW GRANTS FOR 'username'@'localhost';

Table Doesn't Exist

Query refers to a table that doesn't exist in the database

ERROR 1146 (42S02): Table 'database.table' doesn't exist
Solution

1. Check if you're using the correct database:

SHOW DATABASES;

2. Check available tables in the database:

USE database_name; SHOW TABLES;

3. Check for typos in table name or create the table if needed

Too Many Connections

MySQL server has reached the maximum allowed connections

ERROR 1040 (HY000): Too many connections
Solution

1. Check current connection limit and used connections:

SHOW VARIABLES LIKE 'max_connections';

SHOW STATUS LIKE 'Threads_connected';

2. Increase max_connections in my.cnf:

max_connections = 200

3. Identify and terminate unnecessary connections:

SHOW PROCESSLIST;

Deadlock Found

Transaction was rolled back due to a deadlock

ERROR 1213 (40001): Deadlock found when trying to get lock; try restarting transaction
Solution

1. Identify and optimize problematic queries

2. Add retry logic to your application for deadlock scenarios

3. Check deadlock information:

SHOW ENGINE INNODB STATUS;

4. Consider reordering operations in transactions to prevent deadlocks

Database Deadlock in Magento

Database deadlock found when trying to get lock in Magento

SQLSTATE[40001]: Serialization failure: 1213 Deadlock found when trying to get lock; try restarting transaction
Solution

1. Check for long-running queries: SHOW PROCESSLIST;

2. Optimize your database with: bin/magento indexer:reindex

3. Consider increasing innodb_buffer_pool_size in MySQL config

MySQL Wait Timeout in Magento

MySQL server has gone away during Magento operations

SQLSTATE[HY000]: General error: 2006 MySQL server has gone away
Solution

Increase wait_timeout and max_allowed_packet in MySQL:

wait_timeout = 600

max_allowed_packet = 256M

PHP Common Errors

PHP Memory Limit in Magento

Fatal error: Allowed memory size exhausted in Magento operations

PHP Fatal error: Allowed memory size of 756M bytes exhausted (tried to allocate 16 bytes)
Solution

Increase PHP memory limit in php.ini for Magento:

memory_limit = 2G

Or for CLI only:

php -d memory_limit=2G bin/magento [command]

Redis Common Errors

Redis Connection in Magento

Cannot connect to Redis server from Magento

Connection refused to Redis server at 127.0.0.1:6379
Solution

1. Check if Redis is running: systemctl status redis

2. Verify Redis configuration in Magento's env.php

3. Restart Redis service if needed: systemctl restart redis

Redis Out of Memory

Redis server running out of memory

ERROR: Out Of Memory allocating 16384 bytes
Solution

1. Increase maxmemory setting in redis.conf:

maxmemory 1gb

2. Set an eviction policy:

maxmemory-policy allkeys-lru

3. Restart Redis after changes: systemctl restart redis

Nginx Common Errors

502 Bad Gateway Error

Nginx returns 502 Bad Gateway when the PHP-FPM upstream server fails to respond correctly

2023/10/25 12:34:56 [error] 12345#0: *123 upstream sent too big header while reading response header from upstream, client: 127.0.0.1, server: example.com, request: "GET /path/to/page HTTP/1.1", upstream: "fastcgi://unix:/var/run/php-fpm.sock", host: "example.com"
Solution

Increase the buffer size in your Nginx configuration:

fastcgi_buffers 16 256k;

fastcgi_buffer_size 256k;

Add these to your server block in the Nginx configuration file and reload Nginx:

sudo nginx -t && sudo systemctl reload nginx

413 Request Entity Too Large

Nginx limits the size of client request bodies, affecting file uploads

2023/10/25 12:34:56 [error] 12345#0: *123 client intended to send too large body: 10485760 bytes, client: 127.0.0.1, request: "POST /upload HTTP/1.1", host: "example.com"
Solution

Increase the maximum allowed body size in Nginx:

client_max_body_size 100M;

This can be added to the http, server, or location context in your Nginx config

404 Not Found for PHP Files

PHP files are showing 404 errors despite being present

2023/10/25 12:34:56 [error] 12345#0: *123 FastCGI sent in stderr: "Primary script unknown" while reading response header from upstream, client: 127.0.0.1
Solution

Check your fastcgi_param SCRIPT_FILENAME configuration:

fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;

Also verify the root directory is correctly set:

root /var/www/html;

SSL Certificate Issues

SSL handshake failure or certificate errors

2023/10/25 12:34:56 [error] 12345#0: *123 SSL_do_handshake() failed (SSL: error:1408F10B:SSL routines:ssl3_get_record:wrong version number) while SSL handshaking to upstream
Solution

1. Check that SSL certificate and key paths are correct:

ssl_certificate /etc/nginx/ssl/certificate.crt;

ssl_certificate_key /etc/nginx/ssl/private.key;

2. Ensure certificate is valid and not expired:

openssl x509 -in /etc/nginx/ssl/certificate.crt -text -noout

3. Verify certificate chain is complete

Varnish Common Errors

HTTP Header Size Limits

Varnish limits on HTTP headers can cause issues with large cookies or headers

Message from VCC-compiler: Error 400 Bad Request Guru meditation: XID: 123456 Varnish cache server
Solution

Check current header limits:

sudo varnishadm param.show http_max_hdr

sudo varnishadm param.show http_req_hdr_len

sudo varnishadm param.show http_resp_hdr_len

Increase limits by adding these parameters to your Varnish startup configuration:

-p workspace_backend=2097152

-p http_resp_size=1082768

-p http_resp_hdr_len=1050000

-p workspace_client=2097152

-p http_req_size=65536

-p http_req_hdr_len=32768

Backend Fetch Failed

Varnish cannot connect to or get a response from the backend server

Error 503 Backend fetch failed Guru meditation: XID: 987654 Varnish cache server
Solution

1. Check if the backend server is running:

systemctl status nginx (or Apache)

2. Verify backend configuration in Varnish VCL:

backend default { .host = "127.0.0.1"; .port = "8080"; }

3. Increase backend timeout if necessary:

.first_byte_timeout = 300s; # Default is 60s

.between_bytes_timeout = 300s;

Health Check Failed

Backend server fails Varnish health checks

Child (123) said Child (123) said Backend "default" is Sick Child (123) said Backend "default" Still Sick
Solution

1. Configure better health checks in your VCL:

.probe = { .url = "/health.php"; .timeout = 2s; .interval = 5s; .window = 5; .threshold = 3; }

2. Check Varnish logs for specific health check errors:

varnishlog -g raw -q 'Backend_health'

3. Create a simple health check file on your backend server

VCL Compilation Failure

Errors in your VCL configuration prevent Varnish from starting

Message from VCC-compiler: Expected ';' got '}' ('input' Line 42 Pos 1) } --###-- Running VCC-compiler failed, exit 1
Solution

1. Test your VCL syntax before reloading:

varnishd -C -f /etc/varnish/default.vcl

2. Check for syntax errors like missing semicolons, brackets, etc.

3. Validate that subroutines (vcl_recv, vcl_backend_response, etc.) are properly defined

Git Common Errors

Merge Conflict

Git cannot automatically merge changes

CONFLICT (content): Merge conflict in filename.ext Automatic merge failed; fix conflicts and then commit the result.
Solution

1. Open the file with conflicts and look for <<<<<<< HEAD markers

2. Edit the file to resolve conflicts manually

3. Remove conflict markers and save the file

4. Stage the resolved files:

git add filename.ext

5. Complete the merge:

git commit

Failed to Push Changes

Git refuses to push changes to remote repository

error: failed to push some refs to 'https://github.com/username/repo.git' hint: Updates were rejected because the remote contains work that you do not have locally.
Solution

1. Pull the latest changes first:

git pull origin branch-name

2. Resolve any conflicts if they occur

3. Try pushing again:

git push origin branch-name

4. Alternative (if you're sure): Force push (use with caution):

git push --force-with-lease origin branch-name

Detached HEAD State

Working in detached HEAD state, not on any branch

You are in 'detached HEAD' state. You can look around, make experimental changes and commit them...
Solution

To save your work while in detached HEAD state:

1. Create a new branch at current position:

git checkout -b new-branch-name

2. Or to return to a branch without saving work:

git checkout branch-name

Fatal: Not a Git Repository

Attempting to use Git commands outside a Git repository

fatal: not a git repository (or any of the parent directories): .git
Solution

1. Navigate to your Git repository directory

2. Or initialize a new Git repository:

git init

3. Or clone an existing repository:

git clone repository-url

Cannot Delete Branch

Unable to delete a branch due to unmerged changes

error: The branch 'branch-name' is not fully merged. If you are sure you want to delete it, run 'git branch -D branch-name'.
Solution

1. Check if the branch has unmerged commits:

git log main..branch-name --oneline

2. If you're sure you want to delete the branch:

git branch -D branch-name

3. To delete remote branch (if needed):

git push origin --delete branch-name

Large File Rejection

Git refuses to push because file exceeds GitHub's size limit

remote: error: GH001: Large files detected. You may want to try Git Large File Storage - https://git-lfs.github.com. remote: error: Trace: 08b7c5a3233e4a87d007f63f32e94f72 remote: error: See http://git.io/iEPt8g for more information. remote: error: File large-file.zip is 123.45 MB; this exceeds GitHub's file size limit of 100.00 MB
Solution

1. Remove the large file from your repository:

git rm --cached large-file.zip

2. Commit the removal:

git commit -m "Remove large file"

3. Consider using Git LFS for large files:

git lfs install

git lfs track "*.zip"

git add .gitattributes

git add large-file.zip

git commit -m "Add large file via Git LFS"

Authentication Failed

Git cannot authenticate with the remote repository

remote: Invalid username or password. fatal: Authentication failed for 'https://github.com/username/repo.git/'
Solution

1. Check your credentials are correct

2. For GitHub, use a personal access token instead of password:

- Go to GitHub > Settings > Developer Settings > Personal Access Tokens

- Generate a new token with appropriate permissions

- Use the token as your password when Git prompts you

3. Set up SSH authentication:

ssh-keygen -t ed25519 -C "your_email@example.com"

cat ~/.ssh/id_ed25519.pub (add this to GitHub/GitLab)

4. Update remote URL to use SSH:

git remote set-url origin git@github.com:username/repo.git

Undo Last Commit

Need to undo the last commit but preserve the changes

User wants to undo a commit that has already been pushed
Solution

1. To undo last commit but keep changes staged:

git reset --soft HEAD~1

2. To undo last commit and unstage changes:

git reset HEAD~1

3. To completely discard last commit and changes:

git reset --hard HEAD~1

4. If you've already pushed, create a revert commit:

git revert HEAD
